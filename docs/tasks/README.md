# Task Specs

This directory stores implementation-ready task specs for agents.

Naming convention:

- `NNN-short-name.md` (for example `001-uart2-115200-sniffer.md`)
- lower number means higher implementation priority

Each task spec should be decision-complete and include:

- objective
- constraints
- interface/API changes
- test scenarios
- acceptance criteria

Use [`task-template.md`](task-template.md) for new tasks.
